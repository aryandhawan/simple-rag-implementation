# Gradient Descent

Gradient descent is the foundational optimization algorithm used to train virtually all modern machine learning models. Its purpose is to iteratively adjust a model's parameters in order to minimize a loss function — a scalar measure of how far the model's predictions are from the desired outputs. The algorithm works by computing the gradient of the loss with respect to each parameter (a vector pointing in the direction of steepest increase) and taking a small step in the opposite direction, thereby nudging the parameters toward values that reduce the loss.

The update rule for gradient descent is straightforward: for each parameter θ, the new value is θ ← θ − η∇L(θ), where η is the learning rate (a small positive scalar) and ∇L(θ) is the gradient of the loss with respect to θ. The learning rate is among the most critical hyperparameters in training: too large and the updates overshoot minima or cause divergence; too small and training converges very slowly. Practitioners typically start with a reference value — such as 1e-3 for Adam — and adjust it based on the observed training curve.

Batch gradient descent computes the gradient using the entire training dataset before taking a single update step. This produces a precise, low-variance gradient estimate, but it is computationally impractical for large datasets. Stochastic gradient descent (SGD) is the opposite extreme: it estimates the gradient using a single randomly drawn training example per update, making each step very fast but the gradient estimate very noisy. Mini-batch gradient descent — the standard approach in deep learning — computes gradients over a small batch of examples (typically 32 to 512), balancing computational efficiency with gradient quality.

Momentum is an extension of vanilla gradient descent that accumulates a moving average of past gradients and uses this "velocity" to smooth out the optimization trajectory. By dampening oscillations in directions of high curvature and accelerating progress in consistently downhill directions, momentum significantly speeds up convergence. The momentum hyperparameter β (commonly 0.9) controls how much of the previous velocity is retained at each step.

Adaptive learning rate optimizers — most notably Adam (Adaptive Moment Estimation) — combine momentum with per-parameter learning rate adaptation. Adam maintains estimates of both the first moment (mean) and second moment (uncentered variance) of the gradients and uses these to scale the learning rate for each parameter individually. Parameters with historically small or consistent gradients receive larger effective learning rates; parameters with large or noisy gradients receive smaller ones. Adam has become the default optimizer for training large language models.

Learning rate schedules further improve training by varying η over time. Common schedules include linear warm-up, where the learning rate ramps from zero to its peak value over the first several thousand steps (reducing instability early in training), followed by cosine decay, which gradually reduces the learning rate toward zero over the rest of training. The combination of warm-up and cosine decay is nearly universal in Transformer pre-training recipes.
