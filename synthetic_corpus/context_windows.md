# Context Windows

The context window of a language model is the maximum number of tokens the model can attend to simultaneously during a single forward pass. All tokens within the context window — including the system prompt, conversation history, retrieved documents, and the current input — are visible to the model and can influence its output. Tokens outside the context window are simply not accessible to the model without explicit retrieval or summarization. The context window size is therefore one of the most practically important constraints on what a language model can do in a given interaction.

Early Transformer language models had context windows of 512 or 1024 tokens. This was sufficient for sentence-level tasks but too short to process full documents or maintain coherent long conversations. The memory and compute requirements of self-attention, which is quadratic in sequence length, made extending the context window technically expensive. Subsequent research focused on both architectural improvements and positional encoding schemes that generalize to lengths beyond those seen during training.

Rotary Position Embeddings (RoPE) and Alibi (Attention with Linear Biases) are two positional encoding approaches designed to extrapolate more gracefully to longer sequences than sinusoidal or learned absolute positions. Models trained with RoPE can be extended to longer contexts through fine-tuning on longer sequences at a modest additional cost, a technique used to extend models like Llama from 4k to 32k or 128k token contexts. Modern frontier models — including GPT-4, Claude, and Gemini — offer context windows ranging from 128k to 1 million tokens or more.

Despite these advances, effectively using long contexts remains an active research problem. The "lost in the middle" phenomenon, documented empirically, shows that language models tend to attend more strongly to information at the beginning and end of a long context, while information in the middle of a very long context is more likely to be overlooked or forgotten during reasoning. This has implications for how documents should be ordered within a long context and whether RAG retrieval is still preferable to simply stuffing all documents into the context.

Context length also affects inference cost. For autoregressive generation, the KV cache stores key and value tensors for all previous tokens, allowing the model to avoid recomputing them for each new generated token. The KV cache grows linearly with context length, and for very long contexts it can consume as much GPU memory as the model weights themselves. Techniques like sliding window attention, sparse attention, and KV cache compression (evicting or quantizing old cache entries) reduce this memory burden at the cost of some degradation in long-range attention quality.

Prompt design strategies for working within context limits include summarization of earlier conversation turns, selective inclusion of only the most relevant retrieved passages, and hierarchical processing approaches that process long documents in chunks and aggregate the results.
