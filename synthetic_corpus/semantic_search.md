# Semantic Search

Semantic search is a retrieval paradigm that matches documents to queries based on their meaning rather than their literal vocabulary. Traditional keyword-based search engines match documents by counting how many query terms they contain; semantic search instead compares the semantic content of the query and the document, enabling it to retrieve relevant results even when the query and the document use entirely different words to express the same idea. This is particularly valuable for natural language queries where users express their intent in conversational phrasing rather than carefully chosen index terms.

The foundation of modern semantic search is dense retrieval using embedding models. Both the query and each document in the corpus are independently encoded into embedding vectors by a neural network trained to place semantically similar texts near each other in the vector space. A query like "how do I lower my blood pressure naturally?" and a document containing the phrase "non-pharmacological management of hypertension" may have zero vocabulary overlap, yet a well-trained embedding model will place their vectors close together because it has learned that these phrases describe the same concept. At query time, the query is embedded and the nearest document vectors are retrieved using approximate nearest-neighbor search.

The quality of semantic search is tightly coupled to the quality of the embedding model. Models trained on large corpora of paired texts — such as question-passage pairs from web search logs, natural language inference datasets, or question-answer pairs — learn finer semantic distinctions than models trained on unlabeled text alone. Bi-encoder models encode queries and documents separately, enabling pre-computation of document embeddings for efficient search. Cross-encoder models process query and document jointly, enabling more accurate relevance scoring at the cost of speed.

Fine-grained relevance is one of semantic search's key advantages over keyword search. It can handle paraphrase, synonymy, hypernymy (a document about "animals" is relevant to a query about "dogs"), and complex compositional meaning. It also handles typos and informal phrasing more gracefully, because the embedding model maps these variants close to their correctly spelled or formal equivalents in vector space.

Semantic search struggles with queries that rely on exact term matching. A query for a specific product serial number, a rare proper noun, or a technical code that the embedding model has not encountered will not embed in a meaningful location relative to relevant documents. The embedding model's training distribution determines which semantic distinctions it represents accurately, and out-of-distribution vocabulary is poorly handled. This is why hybrid search — combining semantic retrieval with BM25 — is often preferred in production systems.

Evaluation of semantic search systems uses standard information retrieval metrics. Recall at k (R@k) measures what fraction of the relevant documents in the corpus appear in the top-k retrieved results. Mean Reciprocal Rank (MRR) rewards systems that place the first relevant result higher in the ranked list. Normalized Discounted Cumulative Gain (NDCG) accounts for graded relevance, giving more credit for placing highly relevant documents near the top. BEIR (Benchmarking IR) is a widely used suite of heterogeneous retrieval benchmarks for comparing semantic search models across domains.
