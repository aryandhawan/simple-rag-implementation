# Attention Mechanisms

Attention mechanisms allow neural networks to dynamically focus on different parts of their input when producing each element of their output. Rather than compressing an entire input sequence into a single fixed-size vector, attention enables the model to selectively weight the importance of each input element at each step of computation. This idea was first applied to sequence-to-sequence models in 2015 and later became the central building block of the Transformer architecture.

The basic attention operation takes three inputs: queries, keys, and values. The query represents the current element seeking information, the keys represent all the elements being attended to, and the values contain the actual information to be aggregated. Attention scores are computed by measuring the compatibility between each query and each key — typically via a dot product — and then normalizing these scores with a softmax function to produce a probability distribution. The output is a weighted sum of the values, where each value's weight is its normalized attention score.

Scaled dot-product attention, the variant used in Transformers, divides the dot product by the square root of the key dimension before applying softmax. This scaling prevents the dot products from growing too large in magnitude as dimensionality increases, which would push the softmax into regions with very small gradients and slow learning.

Multi-head attention extends this by running several attention operations in parallel, each with its own learned projection matrices for queries, keys, and values. The outputs of all heads are concatenated and linearly projected back to the model's hidden dimension. The motivation is that different heads can specialize in attending to different types of relationships — syntactic dependencies, coreference, positional proximity — simultaneously within a single layer.

Self-attention is the specific case where the queries, keys, and values all derive from the same sequence. This allows every token to directly attend to every other token in the same sequence, capturing long-range dependencies without the vanishing gradient problems that affected RNNs. In contrast, cross-attention uses queries from one sequence (e.g., the decoder's current state) and keys and values from another (e.g., the encoder's output), enabling the model to condition its generation on a separate context.

Causal masking, also called masked attention, is used in autoregressive decoders to prevent each token from attending to future tokens. This is implemented by setting the attention scores for future positions to negative infinity before the softmax, ensuring those positions receive zero weight. Without this masking, the model could trivially "cheat" by reading ahead during training.
