# BM25

BM25 (Best Match 25) is a probabilistic ranking function used in information retrieval to estimate the relevance of documents to a query based on the frequency of query terms in each document. It is the current state-of-the-art sparse retrieval algorithm and has been the default ranking function for major search engines and document retrieval systems for decades. BM25 belongs to the BM (Best Match) family of ranking functions derived from the probabilistic relevance model and extends the simpler TF-IDF scheme with saturation and document length normalization.

The BM25 score for a document d given a query q is computed by summing contributions from each query term: BM25(d, q) = Σ IDF(t) × [f(t, d) × (k1 + 1)] / [f(t, d) + k1 × (1 − b + b × |d| / avgdl)], where f(t, d) is the frequency of term t in document d, |d| is the document length in tokens, avgdl is the average document length in the corpus, and k1 and b are tunable parameters. The IDF (Inverse Document Frequency) component, IDF(t) = log((N − n_t + 0.5) / (n_t + 0.5)), penalizes terms that appear in many documents across the corpus, following the intuition that rare terms are more informative for distinguishing relevant documents.

The k1 parameter controls term frequency saturation. In TF-IDF, relevance increases without bound as a term appears more often in a document. BM25's saturation formula ensures that relevance increases quickly for the first few occurrences of a term but flattens out as frequency grows further — a document is not infinitely more relevant just because a query term appears fifty times rather than five. The value k1 = 1.2 or k1 = 2.0 is commonly used.

The b parameter controls document length normalization. Longer documents have a higher probability of containing any given term simply by virtue of having more words, so raw term frequencies in long documents overstate relevance relative to shorter documents. The normalization factor (1 − b + b × |d| / avgdl) scales down the effective term frequency for long documents when b = 1 and applies no length normalization when b = 0. The standard default is b = 0.75.

BM25 operates on an inverted index: a data structure that, for each term in the vocabulary, stores a list (posting list) of documents containing that term along with their term frequencies. At query time, the system looks up the posting lists for all query terms and computes the BM25 score for the union of matching documents. This can be done extremely efficiently, making BM25 fast enough to search millions of documents in milliseconds.

Despite being a decades-old algorithm, BM25 remains a competitive baseline and an indispensable component of hybrid search systems. Its strength lies in exact keyword matching, making it particularly effective for queries containing rare terms, technical vocabulary, or named entities that semantic search models may represent poorly. In practice, combining BM25 with dense retrieval through hybrid search consistently outperforms either method alone across a wide range of query types and domains.
