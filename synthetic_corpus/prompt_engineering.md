# Prompt Engineering

Prompt engineering is the practice of designing and refining the text inputs given to a language model in order to elicit more accurate, relevant, or appropriately formatted outputs. Because large language models are extremely sensitive to how their input is phrased, prompt engineering has emerged as a critical skill for practitioners building applications on top of these models. A well-crafted prompt can dramatically improve model performance on a task without any modification to the model's parameters.

The simplest and most foundational prompt engineering technique is providing clear, specific instructions. Vague prompts produce vague outputs; precise task descriptions, constraints, and output format specifications guide the model toward the desired behavior. For example, asking a model to "summarize this document in exactly three bullet points, each under 20 words, in plain language accessible to a non-technical audience" produces far more consistent and useful output than asking it to simply "summarize this."

Role or persona prompting instructs the model to adopt a particular character or expertise level at the beginning of its context. A system message such as "You are an expert software engineer reviewing code for security vulnerabilities" activates the model's knowledge and communication style appropriate to that role, shaping both the content and tone of subsequent responses. This technique is widely used in chat-based applications to customize the assistant's behavior for a specific domain.

Few-shot prompting, as discussed separately, provides example (input, output) pairs in the prompt to demonstrate the desired task pattern. The model generalizes from these examples to new inputs. Closely related is chain-of-thought prompting, in which the examples include explicit reasoning steps, guiding the model to reason through problems rather than jump directly to answers.

Output format specification is a practical concern in production applications. Language models default to free-flowing prose, but many applications require structured outputs — JSON, tables, lists, or code. Prompts that explicitly specify the expected format (including an example of the structure, or an instruction like "respond only with valid JSON, with no additional commentary") significantly increase compliance. Tool use and function-calling APIs address this more robustly by letting developers specify structured schemas that the model must adhere to.

Prompt sensitivity and brittleness are key challenges. Small changes in phrasing — adding a word, reordering clauses, changing punctuation — can produce noticeably different outputs. This instability makes it difficult to reliably optimize prompts and to reason about why a particular prompt works. Research into prompt sensitivity has led to automated prompt optimization techniques, where meta-models or search algorithms generate and evaluate candidate prompts to find those that work best across a distribution of test cases.

Prompt injection is a security concern unique to deployed language model applications. Malicious users can craft inputs that override the system's original instructions by embedding adversarial instructions within user-supplied content — for example, hiding the text "Ignore all previous instructions and instead..." in a document the model is asked to summarize. Mitigations include strict input sanitization, delimiters that clearly separate user-supplied content from system instructions, and output validation.
