# Mixture of Experts

Mixture of Experts (MoE) is a neural network architecture that replaces dense layers with a collection of parallel sub-networks called experts, combined with a learned routing mechanism that selects which experts to activate for each input. The central appeal of MoE is that it allows the total number of parameters in a model to grow much larger than the number that participates in any single forward pass, enabling vastly larger models without a proportional increase in inference compute.

In a standard Transformer with MoE, the feed-forward sub-layer in each Transformer block is replaced by an MoE layer. An MoE layer consists of N expert networks — each structurally identical to the dense feed-forward layer it replaces — and a gating network (router). For each input token, the router computes a score for each expert, selects the top-k experts (typically k = 1 or k = 2), and routes the token's representation to those experts. The outputs are combined as a weighted sum using the router's softmax scores before being passed to the next layer.

Sparse routing is what makes MoE computationally efficient. If a layer has 64 experts and routes each token to 2 of them, only about 3% of the expert parameters are active per token. The model's total parameter count reflects all 64 experts, but the floating-point operations per forward pass reflect only the 2 activated experts. This separation of parameter count from compute allows practitioners to scale models to trillions of parameters — as in Google's Switch Transformer and Mixtral from Mistral AI — while keeping inference costs comparable to a much smaller dense model.

Load balancing is a critical challenge in training MoE models. Without special measures, the router tends to collapse: a small subset of experts receives most of the tokens, while other experts receive little training signal and become underutilized. An auxiliary load-balancing loss penalizes uneven expert utilization by encouraging the router to distribute tokens more evenly across experts. Expert capacity limits, which drop tokens that exceed a fixed per-expert budget, also prevent a single expert from being overwhelmed at the cost of occasionally discarding tokens.

Expert specialization is an empirically observed property of well-trained MoE models: different experts tend to develop distinct functional specializations, handling particular token types, linguistic phenomena, or domains. Analysis of routing decisions shows that some experts activate primarily for numeric tokens, others for punctuation, and others for domain-specific vocabulary, though this specialization is soft and not rigidly enforced by the architecture.

MoE models present unique challenges for deployment. Because all expert weights must reside in memory even when most are inactive, serving a sparse MoE model requires more memory than a dense model with the same active-parameter count. Distributed serving systems must route token activations across the GPUs or TPUs that hold different experts, introducing communication overhead that can partially offset the compute savings.
