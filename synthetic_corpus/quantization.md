# Quantization

Quantization is a model compression technique that reduces the numerical precision used to store and compute with a model's weights and activations. Standard deep learning models are trained using 32-bit floating-point (FP32) numbers, but inference and sometimes training can be performed with lower-precision formats such as 16-bit float (FP16 or BF16), 8-bit integers (INT8), or even 4-bit integers (INT4). The primary benefits are a reduction in memory footprint, faster arithmetic on hardware that supports lower-precision operations, and lower power consumption.

Post-training quantization (PTQ) applies quantization to a fully trained model without any additional training. The simplest form, weight-only quantization, converts just the model's parameters to a lower-precision format while keeping activations in full or half precision during computation. Activation quantization also converts intermediate tensors, which is more aggressive but requires careful calibration to avoid large accuracy losses. PTQ typically requires only a small calibration dataset to determine the appropriate quantization ranges for each tensor.

Quantization-aware training (QAT) simulates the effects of quantization during the training process itself by introducing "fake quantization" operations — quantizing and then immediately dequantizing values — into the forward pass. This allows the model's weights to be optimized knowing that they will be quantized, which typically produces higher accuracy than PTQ at the same bit width. QAT is more expensive because it requires a full training run, but it is valuable for aggressive quantization levels such as INT4 or INT2.

The key quality measure in quantization is how much the model's accuracy degrades compared to the full-precision baseline. This degradation depends on the bit width, the quantization scheme (symmetric vs. asymmetric, per-tensor vs. per-channel), and the model architecture. Modern large language models often tolerate INT8 weight quantization with negligible accuracy loss, while INT4 quantization introduces a small but measurable degradation that may be acceptable given the memory savings.

GPTQ is a widely used algorithm for post-training quantization of large language models. It quantizes weights layer by layer using a second-order optimization approach, updating the remaining unquantized weights to compensate for the error introduced by quantizing each weight. This iterative compensation allows GPTQ to achieve INT4 quantization with better accuracy than naive rounding. Similarly, AWQ (Activation-aware Weight Quantization) identifies and protects the weights that most strongly influence the model's activations, applying quantization more carefully to the most important channels.

QLoRA combines quantization with low-rank adaptation to enable fine-tuning of large models on consumer hardware. The base model is loaded in 4-bit quantized form using a format called NF4 (Normal Float 4), which is designed to match the distribution of normally distributed weights. LoRA adapters are then trained in 16-bit precision on top of the frozen quantized base, achieving fine-tuning quality competitive with full-precision methods at a fraction of the memory cost.
