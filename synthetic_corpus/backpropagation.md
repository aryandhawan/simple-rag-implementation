# Backpropagation

Backpropagation is the algorithm used to compute the gradient of a loss function with respect to every parameter in a neural network. It is the computational engine that makes gradient descent feasible for deep networks: rather than separately perturbing each parameter to estimate its effect on the loss — which would require as many forward passes as there are parameters — backpropagation computes all gradients in a single backward pass through the network using the chain rule of calculus.

The chain rule states that the derivative of a composite function can be computed by multiplying the derivatives of its constituent functions in sequence. A neural network is, by construction, a composition of many parametric functions stacked together, so the chain rule applies directly. To find how much the loss L changes when a parameter in the first layer is adjusted, backpropagation multiplies together the local derivatives at every layer the signal must pass through, from the loss back to that parameter.

The backward pass begins at the output layer. Starting from the derivative of the loss with respect to the network's final output (∂L/∂ŷ), the algorithm propagates error signals layer by layer toward the input. At each layer, it computes two quantities: the gradient with respect to the layer's parameters (needed to update those parameters) and the gradient with respect to the layer's input (needed to continue propagating the error to earlier layers). These two computations exploit the intermediate values computed during the forward pass, which is why neural network frameworks store these values in memory during the forward pass — a technique called caching or the "forward graph."

In practice, backpropagation is implemented through automatic differentiation libraries such as PyTorch's Autograd or TensorFlow's tape-based gradient system. These libraries construct a computational graph during the forward pass that records every operation applied to the model's parameters. The backward pass then traverses this graph in reverse, automatically applying the chain rule to each recorded operation. Users generally do not implement backpropagation manually; they simply call loss.backward(), and the library handles the rest.

Vanishing and exploding gradients are classical failure modes in deep networks. During backpropagation, if the local gradients at each layer are consistently smaller than one, their product across many layers approaches zero, making the gradients of early-layer parameters negligibly small and preventing those parameters from learning. If the local gradients are consistently larger than one, the product grows exponentially, causing numerical instability. Techniques like careful weight initialization (e.g., Xavier/He initialization), batch normalization, residual connections, and gradient clipping address these problems in modern architectures.

Computational cost of backpropagation is roughly two to three times that of a forward pass, because both the forward activations (for caching) and the backward gradient computations must be performed. For very large models with long sequences, this memory cost — storing all intermediate activations — can become the bottleneck. Gradient checkpointing addresses this by recomputing activations during the backward pass rather than storing them all, trading compute for memory.
