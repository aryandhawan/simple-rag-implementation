# Vector Databases

A vector database is a specialized data storage system designed to index, store, and efficiently query high-dimensional embedding vectors. Unlike traditional relational databases that organize data into rows and columns and retrieve records by exact key matches, vector databases are optimized for similarity search: given a query vector, they return the database entries whose vectors are most similar, typically measured by cosine similarity, dot product, or Euclidean distance. This capability is fundamental to applications such as semantic search, retrieval-augmented generation, recommendation systems, and image similarity search.

The central challenge that vector databases solve is approximate nearest-neighbor (ANN) search at scale. Brute-force comparison of a query vector against every stored vector is exact but computationally prohibitive for large collections — a corpus of ten million vectors each with 1536 dimensions would require billions of floating-point operations per query. ANN algorithms trade a small, controlled amount of accuracy for dramatically faster query times. The most widely used indexing structure is HNSW (Hierarchical Navigable Small World), which organizes vectors into a layered graph that supports logarithmic-time approximate search.

HNSW builds a multi-layer graph where each layer contains a subset of the vectors. The top layer has the fewest nodes and represents the coarsest structure; lower layers become progressively denser. A query starts at the top layer, greedily navigates to the nearest neighbor at each layer, then descends to the next layer and repeats. This hierarchical traversal quickly narrows the search to a small neighborhood in the densest bottom layer, where the final approximate nearest neighbors are identified. HNSW parameters like the maximum number of connections per node (M) and the construction-time search depth (ef_construction) control the trade-off between index build time, memory usage, query speed, and recall.

IVF (Inverted File) indexing is another popular approach, borrowed from information retrieval. The vector space is partitioned into clusters using k-means, and each vector is assigned to its nearest cluster centroid. At query time, only the vectors in the few nearest clusters are searched, reducing the comparison set from the full corpus to a small fraction. IVF is often combined with product quantization (IVF-PQ), which compresses each vector by splitting it into sub-vectors and encoding each with a compact codebook. This dramatically reduces memory requirements at the cost of some recall.

Prominent vector database systems include Pinecone, a managed cloud service; Weaviate, which combines vector search with structured metadata filtering; Qdrant, which emphasizes high-performance Rust-based internals; Chroma, designed for RAG and AI application development; and Milvus, an open-source system with a rich feature set. Facebook's Faiss library is a widely used non-database toolkit for building ANN indices. Many traditional databases — including Postgres with the pgvector extension, Elasticsearch, and Redis — have added vector search capabilities to their existing feature sets.

Filtering is a common requirement in production vector search: users often want to find the most similar vectors that also satisfy metadata constraints, such as documents from a particular date range or a specific category. Handling filters efficiently without sacrificing recall is a non-trivial engineering challenge, and different systems address it through pre-filtering, post-filtering, or hybrid index structures.
