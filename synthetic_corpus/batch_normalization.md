# Batch Normalization

Batch normalization is a technique for stabilizing and accelerating the training of deep neural networks by normalizing the activations within each layer across a mini-batch of training examples. Introduced by Ioffe and Szegedy in 2015, it addresses the problem of internal covariate shift — the phenomenon in which the distribution of a layer's inputs changes as the parameters of the preceding layers are updated during training, forcing each layer to continually adapt to new input statistics and slowing convergence.

The operation is applied to each feature dimension independently. For a mini-batch of m examples, batch normalization computes the mean and variance of each activation across the batch dimension, then normalizes the activations to have zero mean and unit variance. Two learned parameters — a scale γ and a shift β — are then applied to the normalized activations, allowing the network to recover any mean or variance it determines to be optimal rather than being locked to zero mean and unit variance. The forward pass computation is: x̂ = (x − μ_B) / √(σ²_B + ε), followed by y = γx̂ + β, where μ_B and σ²_B are the batch mean and variance, and ε is a small constant for numerical stability.

The practical effects of batch normalization are substantial. Networks trained with it can use higher learning rates without destabilizing, because the normalized activations make the optimization landscape smoother. It also acts as a regularizer: because the mean and variance are computed from a random mini-batch, each training example is normalized by slightly different statistics at each step, introducing noise that reduces overfitting in a manner similar to dropout. This regularization effect often allows practitioners to reduce or eliminate explicit dropout when batch normalization is used.

Batch normalization has important limitations related to its dependence on the batch dimension. At inference time, the batch typically contains a single example, so there is no mini-batch over which to compute statistics. This is addressed by maintaining running estimates of the mean and variance during training (exponential moving averages) and using these fixed estimates at inference time. Additionally, batch normalization is inappropriate for sequence models like recurrent neural networks, where the batch statistics mix information from different positions in the sequence.

Layer normalization is a popular alternative that normalizes across the feature dimension rather than the batch dimension, computing mean and variance for each individual example independently. Because it does not depend on batch statistics, layer normalization works correctly for any batch size, including batch size 1, and handles variable-length sequences naturally. It has become the default normalization technique in Transformer models, while batch normalization remains prevalent in convolutional networks for image processing.

Group normalization divides the feature channels into groups and normalizes within each group, providing a middle ground between layer normalization and batch normalization that works well for small batch sizes in image segmentation and detection tasks.
