# Cross-Entropy Loss

Cross-entropy loss is the most widely used training objective in classification tasks and language modeling. It measures the difference between two probability distributions: the model's predicted probability distribution over possible outputs and the true distribution represented by the ground-truth labels. Minimizing cross-entropy loss encourages the model to assign high probability to the correct class and low probability to all others, directly optimizing the model's predictive accuracy in a probabilistic sense.

The formula for cross-entropy loss on a single example is: L = −Σ y_c × log(p_c), where the sum is over all possible classes c, y_c is 1 for the true class and 0 for all others (the one-hot label), and p_c is the model's predicted probability for class c. Because only the true class has a nonzero label, this simplifies to L = −log(p_true), the negative log-probability of the correct class. Minimizing this loss is equivalent to maximizing the log-likelihood of the correct answer under the model's distribution.

In language modeling, cross-entropy loss is computed over the entire vocabulary at each position in the sequence. The model predicts a probability distribution over all tokens for the next position, and the loss is the negative log-probability assigned to the actual next token. Averaged over all positions in the training corpus, this is the standard next-token prediction objective used to pre-train GPT-style models. A model that perfectly memorized its training corpus would assign probability 1.0 to each correct next token, achieving a cross-entropy loss of zero.

Perplexity is a related metric derived from cross-entropy: perplexity = e^(average cross-entropy loss), equivalently 2^(average cross-entropy in bits). It measures how "surprised" the model is on average by the next token. A perplexity of 20 means the model is, on average, as uncertain as if choosing uniformly among 20 equally likely tokens. Lower perplexity indicates better language modeling, and it is used to compare language models trained on the same test corpus.

The cross-entropy loss has a natural probabilistic interpretation through maximum likelihood estimation. Training a model by minimizing cross-entropy loss on labeled data is equivalent to maximizing the log-likelihood of the training data under the model — a well-established statistical principle. This equivalence connects deep learning training to classical statistical theory and provides theoretical justification for the loss function's widespread use.

Label smoothing is a common modification to cross-entropy training that replaces the one-hot target distribution with a smoothed version: instead of placing all probability mass on the true class, it assigns probability (1 − ε) to the correct class and distributes ε evenly across all other classes. This prevents the model from becoming overconfident — assigning probability very close to 1.0 to its predictions — and improves generalization and calibration. Label smoothing with ε = 0.1 is a standard hyperparameter in Transformer training recipes.
