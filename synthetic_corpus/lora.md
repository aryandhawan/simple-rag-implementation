# LoRA

Low-Rank Adaptation (LoRA) is a parameter-efficient fine-tuning technique that reduces the memory and compute cost of adapting large pre-trained language models to new tasks. Instead of updating all of a model's parameters during fine-tuning, LoRA freezes the original pre-trained weights and injects small trainable matrices into specific layers. Because these additional matrices are constrained to have low rank, the number of trainable parameters is a tiny fraction of the full model size, yet the technique achieves performance competitive with full fine-tuning on many benchmarks.

The mathematical foundation of LoRA rests on the hypothesis that the weight updates needed for task adaptation have a low intrinsic rank. For a weight matrix W of shape (d, k), LoRA represents the update ΔW as the product of two smaller matrices: A of shape (d, r) and B of shape (r, k), where r is the chosen rank and r ≪ min(d, k). During training, only A and B are updated; the original W remains frozen. At inference time, the adapted weight is simply W + AB, so no additional latency is introduced compared to the original model.

LoRA is typically applied to the query and value projection matrices of the attention layers, though it can be extended to any linear layer including the feed-forward sub-network. The rank r is a key hyperparameter: lower ranks reduce memory further but may limit expressive capacity, while higher ranks increase the number of trainable parameters. Common choices in practice range from r = 4 to r = 64. An accompanying scaling hyperparameter, alpha (α), controls the magnitude of the LoRA update relative to the original weights.

The memory savings from LoRA are substantial. Fine-tuning a 7-billion-parameter model with full precision might require four to eight high-memory GPUs, while LoRA fine-tuning of the same model can often fit on a single consumer GPU when combined with quantization techniques like QLoRA. QLoRA loads the frozen base model in 4-bit quantized form and applies LoRA on top, reducing the base model's memory footprint by roughly four times compared to 16-bit precision.

After training, LoRA adapters can be merged back into the base model weights by computing W + AB and storing the result, producing a model with no extra inference overhead. Alternatively, the adapters can be kept separate and swapped dynamically, allowing a single base model to serve multiple specialized tasks by loading different adapter sets without reloading the full model. This composability makes LoRA particularly attractive for production deployments with multiple fine-tuned variants.

LoRA has spawned several variants, including DoRA (Weight-Decomposed Low-Rank Adaptation), which decomposes the weight update into magnitude and direction components, and AdaLoRA, which adaptively allocates the rank budget across different layers based on their importance to the task.
