# Reranking

Reranking is a second-stage retrieval step in which an initial set of candidate documents — retrieved quickly by a first-stage system such as a vector similarity search or BM25 — is scored more carefully and reordered based on a more accurate relevance model. The motivation for a two-stage approach is a fundamental trade-off between speed and precision: the first stage retrieves a large candidate pool rapidly using approximate or lightweight methods, while the reranker applies a more expensive but more accurate model to a smaller subset, balancing overall system efficiency with retrieval quality.

The most common reranking architecture uses a cross-encoder neural network. Unlike the bi-encoder architecture used for dense retrieval — where query and document are encoded independently and compared via dot product — a cross-encoder takes the query and a candidate document concatenated together as a single input and produces a relevance score. Because the query and document are processed jointly, the cross-encoder can model fine-grained interactions between them: whether a specific phrase in the document answers the query's implicit question, whether the document addresses the query's precise intent rather than just sharing vocabulary, and whether the document's context supports a relevant answer.

This joint encoding is more accurate but significantly slower. A dense retrieval bi-encoder can embed a query in milliseconds and compare it against millions of pre-computed document embeddings through ANN search. A cross-encoder must perform a fresh forward pass for each query-document pair at query time, which limits it to scoring a few hundred candidates at most in latency-sensitive applications. This is precisely why reranking is applied only to the top-k candidates from the first stage, typically k in the range of 20 to 100.

ColBERT is a late interaction model that occupies the middle ground between bi-encoders and cross-encoders. It encodes the query and document independently into sequences of token-level embeddings (rather than a single summary embedding), and the relevance score is computed as the sum of maximum inner products between query token embeddings and document token embeddings. This "MaxSim" operation is cheaper than a cross-encoder forward pass, more accurate than a bi-encoder dot product, and amenable to pre-computation of document embeddings.

In retrieval-augmented generation pipelines, reranking is a critical quality gate. After retrieving the top-50 passages with a fast embedding model, a cross-encoder reranker identifies the 5 most truly relevant passages to include in the language model's context. This significantly improves the precision of the context fed to the generator, reducing noise and hallucination caused by marginally relevant or misleading retrieved passages.

Popular reranking models include Cohere's rerank API, BGE-reranker from BAAI, and Jina reranker. These models are typically smaller than the main language model in the pipeline but are specialized through training on query-passage relevance datasets such as MS MARCO.
